2. それぞれのケースのコードの AST を見ながら、ルールを実装する

</div>

<div class="flex justify-around small-code-json">

```json{10,11,15}
// ✅ 正常としたいコードのAST
{
  "body": [
    {
      "type": "TSInterfaceDeclaration",
      "body": {
        "type": "TSInterfaceBody",
        "body": [
          {
            "type": "TSPropertySignature",
            "readonly": true,
            "key": {
              "type": "Identifier",
              "decorators": [],
              "name": "validProperty",
              "optional": false
            },
            "computed": false,
            "optional": false,
            "static": false,
            // ...
```

<div>

```json{10,11,15}
// ❌ 異常としたいコードのAST
{
  "body": [
    {
      "type": "TSInterfaceDeclaration",
      "body": {
        "type": "TSInterfaceBody",
        "body": [
          {
            "type": "TSPropertySignature",
            "readonly": false,
            "key": {
              "type": "Identifier",
              "decorators": [],
              "name": "invalidProperty",
              "optional": false
            },
            "computed": false,
            "optional": false,
            "static": false,
            // ...
```

</div>

</div>

<!-- 
この AST を見ると、TSPropertySignature というタイプのノードには、 readonly というプロパティがあることがわかります。  
つまり、この readonly プロパティが true の場合は正常とし、false の場合は異常とするといった実装内容になりそうです。
-->
